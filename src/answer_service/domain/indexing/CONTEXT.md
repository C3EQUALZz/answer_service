# Indexing — Glossary

The Indexing context owns the catalog of question-answer records and the
process that keeps the search stores in step with the source file.

## Terms

- **QA Pair** — a single question-answer record: the unit that is synchronized
  and indexed for search. Identified by its **External Id**. Aggregate root.

- **External Id** — the stable, source-provided unique identifier of a QA pair.
  It is the identity that crosses into the Search and Conversation contexts. Not
  generated by the system; it comes from the source file and never changes.

- **QA Content** — the change-tracked payload of a QA pair: its question, answer
  and category taken together. Two pairs with equal content are interchangeable
  for indexing purposes.

- **Content Hash** — the fingerprint of a QA Content. Change detection compares
  fingerprints, so re-applying identical content is a no-op. The fingerprint,
  not the source timestamp, decides whether a pair *changed*.

- **Category** — the topic a QA pair belongs to; also usable as a search filter.

- **Source Updated At** — the last-modified timestamp of the record *in the
  source file*. Informational; it does not by itself trigger an update (see
  Content Hash).

- **Indexing Task** — one synchronization run of the source file against the
  search stores. Aggregate root. Moves through **Queued → Running → Succeeded |
  Failed**; a terminal task never transitions again.

- **Sync Stats** — the outcome counters of an Indexing Task: how many pairs were
  **created**, **updated**, **deleted** and **skipped** (skipped = present but
  unchanged).

- **Failure Info** — the structured reason an Indexing Task ended unsuccessfully
  (a code plus a human-readable message).

## Rules

- Synchronization is **idempotent**: running it twice over an unchanged file
  produces the same catalog and only *skipped* counts.
- A pair absent from the source file but present in the catalog is **deleted**
  from the search stores.
- Change is detected by **Content Hash**, independently of Source Updated At.
